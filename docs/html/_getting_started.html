This library enables display and management of local (in-app) notifications.

Local notifications are managed by your application, and not the operating system, unlike Windows toast notifications. They therefore do not appear in the action center.


Installation

The preferred method of installing LocalNotifications is via Nuget. You can either search for and install the package graphically via the NuGet gallery within Visual Studio, or run the following command in the Package Manager Console,

Install-Package RavinduL.LocalNotifications

Alternatively, you could clone or download the GitHub repository and reference the project file located at the path 'src / RavinduL.LocalNotifications / RavinduL.LocalNotifications.csproj' in your Universal Windows project.

Demonstrative Application

An application (pictured on the home page) that demonstrates the functionality of the library is included in the GitHub repository, located at the path 'src/ RavinduL.LocalNotifications.Demo/'. To run this,

  1. Open Visual Studio solution file located at the path 'src/ RavinduL.LocalNotifications.sln'.
  2. Set the 'RavinduL.LocalNotifications.Demo' project as the startup project (an option found within the context menu of associated item within the Solution Explorer).
  3. Ensure that the 'Build' and 'Deploy' options for the app are enabled within the Configuration Manager.
  4. Debug (F5) or Run (Ctrl + F5) the project.

How it Works

Showing a local notification requires two components,

  1. a presenter, and...
  2. a manager.

A presenter is the actual notification that the user sees. It is a templated control containing the definition of what the notification looks and how it behaves (in the form of visual elements, animations, etc...), and the logic for it to be shown, hidden and do whatever else it does. It derives from the LocalNotificationPresenter class, that is a sub-class of the XAML Control class. Presenters provided with the library are defined within the Presenters namespace.

A manager (an instance of the LocalNotificationManager class) is responsible for adding and removing presenters to and from the XAML visual tree as needed. It 'latches' on to a Grid element for this purpose.


Usage

Step 1: Create a Grid element to be assigned to the manager.

<Grid x:Name="NotificationGrid" />

In order to ensure that the local notifications stay above other elements, define the Grid towards the bottom of the XAML file (thereby positioning it above the elements defined physically above it in the markup, within the common parent element). For example,

<Page ...>
    <Grid Background="{ThemeResource ApplicationPageBackgroundThemeBrush}">
 
        <!-- Elements *overlapped by* NotificationGrid (and thereby, local notifications) go above this comment. -->
        <Grid x:Name="NotificationGrid" />
    </Grid>
</Page>

Step 2: Create a manager and assign the newly created Grid to it.

public sealed partial class MainPage : Page
{
    private LocalNotificationManager manager;
 
    public MainPage()
    {
        InitializeComponent();
    }
 
    protected override void OnNavigatedTo(NavigationEventArgs e)
    {
        manager = new LocalNotificationManager(NotificationGrid);
    }   
}

In this case, a field has been declared within the Page, and instantiated as the manager when the page is navigated to.

Step 3: Display a presenter via the manager.

The presenter used for the purpose of this demonstration would be the SimpleNotificationPresenter, and implementation that is provided by default with the library.

Whenever a notification is needed to be shown from the manager (in this case, in the click event handler of a button), the LocalNotificationManager.Show method can be invoked. An instance of a presenter is a required parameter of this method.

private void Button_Click(object sender, RoutedEventArgs e)
{
    var presenter = new SimpleNotificationPresenter
    (
        TimeSpan.FromSeconds(2),
        "Notification text",
        async () => await new MessageDialog("Notification activated.").ShowAsync()
    )
    {
        Background = new SolidColorBrush(Colors.DarkGreen),
    };
 
    manager.Show(presenter);
}

The LocalNotificationManager.Show method accepts an optional parameter of the type LocalNotificationCollisionBehaviour. This defines how a new notification will behave when there already is one visible.
